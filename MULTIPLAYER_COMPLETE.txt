================================================================================
                           HOW IT ALL WORKS
================================================================================

HOSTING:
  1. Player clicks "Host Game"
  2. Server starts listening on port 9999
  3. Your player spawns at spawn point
  4. Game waits for clients to connect

JOINING:
  1. Player clicks "Join Game"
  2. Enters server IP (or localhost for testing)
  3. Connects to server
  4. Server spawns your player
  5. You receive sync updates from other players

REAL-TIME SYNC:
  - Your player: You control locally, syncs every 0.1 seconds
  - Others: Receives RPC updates, smoothly interpolates
  - Input: Only processes if you own the character
  - Movement: Instant for you, smooth for others

AUTHORITY SYSTEM:
  - Each player has authority over themselves
  - You can only control your own character
  - Others see your movement via network sync
  - Only you see instant movement (cheating prevention)

================================================================================
                        NETWORK PERFORMANCE
================================================================================

BANDWIDTH: ~400 bytes/sec per connected player
  Example 4 players: ~1.2 KB/sec (very efficient!)

LATENCY TOLERANCE:
  < 50ms:    Excellent (feels instant)
  50-100ms:  Good (smooth)
  100-200ms: Acceptable (noticeable delay)
  200-500ms: Sluggish (playable)
  > 500ms:   Very jerky (hard to play)

SYNC UPDATES:
  Rate: 10 Hz (every 0.1 seconds) by default
  Type: Position + Rotation
  Method: Unreliable RPC (fast, may drop 1-2%)
  Interpolation: Smooth client-side lerp

================================================================================
                            KEY CONCEPTS
================================================================================

PEER ID:
  - Server: peer ID 1
  - First client: peer ID 2
  - Second client: peer ID 3
  - etc.

PLAYER NAMING:
  - Players named "Player_<peer_id>"
  - Example: "Player_1", "Player_2"

AUTHORITY:
  - Each player has authority (ownership) over themselves
  - Example: Only peer 2 can control Player_2
  - If not authority: Input disabled, gets RPC updates

RPC METHODS:
  - spawn_player() - Create new player (reliable)
  - sync_transform() - Update position/rotation (unreliable, fast)

MULTIPLAYER GROUP:
  - All players added to "players" group
  - Use get_tree().get_nodes_in_group("players") to find all players
